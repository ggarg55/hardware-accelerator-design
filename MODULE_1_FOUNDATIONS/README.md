# Module 1: Foundations of AI Hardware

> **Why do we need custom hardware for AI? What's wrong with CPUs and GPUs?**

This module answers the fundamental "why" question and builds the conceptual foundation you need before diving into hardware design. You'll understand the computing landscape, the AI/ML taxonomy, and see your first examples of turning trained ML models into silicon.

---

## Learning Objectives

By the end of this module, you will be able to:
- Explain why CPUs, GPUs, FPGAs, and ASICs exist and when to use each
- Navigate the AI → ML → DL → LLM hierarchy and its hardware implications
- Design simple hardware circuits for Decision Trees and K-Means classifiers
- Distinguish between training (software) and inference (hardware) phases
- Use fundamental metrics: TOPS, GOPS/Watt, FPS, latency
